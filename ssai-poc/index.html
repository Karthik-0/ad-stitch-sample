How It Works
  1. Click New Session — the server creates a session and fetches the pre-roll ad.
  2. Click Load Session Stream — player opens the personalised master playlist.
  3. The pre-roll plays first, then transitions seamlessly to live content.
  4. During live playback, set a delay and click Insert Ad Break. The stitcher will inject the ad approximately that many seconds from now.
  5. The player never stops — ad segments are stitched server-side; the player sees a continuous stream.
Test with Chrome + hls.js or Safari native HLS. For diagnostics, open browser DevTools → Network and filter for .m3u8.
